# FIRESCUE 裝備配發與 AI 測量系統 - 系統部署與轉移手冊

本手冊提供將本系統（包含前端網頁與後端資料庫）完整轉移至**另一個 Google 帳號**以及**其他主機**的標準作業流程 (SOP)。

---

## 階段一：建立新的 Google 雲端環境 (後端轉移)

請使用「新的 Google 帳號」登入並執行以下步驟：

### 1. 建立照片儲存資料夾
1. 在 Google 雲端硬碟中，建立一個新的資料夾（例如命名為 `FIRESCUE_Photos`）。
2. 對該資料夾點選右鍵 > **共用** > 將一般存取權設為「**知道連結的任何人皆可查看**」。
3. 複製該資料夾的網址，從網址中擷取 **資料夾 ID**。
   > [!TIP]
   > 網址格式為 `https://drive.google.com/drive/folders/【這一段就是資料夾ID】`，請先將這串 ID 記下來。

### 2. 建立新的 Google 試算表 (資料庫)
1. 在雲端硬碟中建立一個新的 Google 試算表。
2. 根據舊有系統的格式，建立以下工作表名稱 (Sheet names)：
   - `系統設定`
   - `分隊箱狀態`
   - `單位資料`
   - `人員名冊`
   - `尺寸參數`
   - `測量紀錄` (可留空，系統會自動寫入)
3. 將舊系統的 `尺寸參數`、`單位資料` 等基礎設定資料複製貼上至新表中。

### 3. 部署 Google Apps Script (GAS API)
1. 在剛剛建立的試算表中，點選上方選單 **「擴充功能」 > 「Apps Script」**。
2. 將本專案完整的 GAS 程式碼貼上覆蓋。
3. **🚨 關鍵修改：** 在程式碼中尋找 `doPost` 函式內的 `var folderId = "..."` 這一行（約在程式碼最下方處）。
   ```javascript
   // 將雙引號內的 ID 替換為您在「步驟 1」取得的新資料夾 ID
   var folderId = "您剛才複製的新資料夾ID";
   ```
4. 儲存程式碼 (Ctrl+S)。
5. 點選右上角 **「部署」 > 「新增部署作業」**。
   - 類型選擇：**網頁應用程式 (Web App)**
   - 執行身分：**我 (您的 Google 帳號)**
   - 誰可以存取：**所有人 (Anyone)**
6. 點擊「部署」，系統會要求授權，請點擊「核准存取權」並完成登入授權（若出現警告提示，請點擊「進階」>「前往不安全的頁面」）。
7. 部署完成後，會產生一段 **網頁應用程式網址 (Web App URL)**，請將此網址複製記下，這就是您的**新 API 網址**。

---

## 階段二：設定並發布前端網頁 (主機轉移)

前端網頁不依賴任何伺服器語言（如 PHP、Node.js），它是一個純靜態網頁（Single Page Application），因此可以放置在 **任何靜態網頁主機**（例如 GitHub Pages、Vercel、Zeabur、AWS S3 或是自己的 Linux/Windows 主機）。

### 1. 修改 `index.html` 內的 API 網址
1. 用文字編輯器 (VS Code 或記事本) 打開 `index.html`。
2. 在大約第 3752 行（`<script>` 區域的最開頭處），找到以下這行：
   ```javascript
   const APPS_SCRIPT_URL = "https://script.google.com/macros/s/AKfycb.../exec";
   ```
3. 將雙引號內的網址，替換為您在**階段一、步驟 3** 取得的 **新 API 網址**。

### 2. 部署至新主機
1. 只需要將修改好的 `index.html` 上傳至您的新網頁主機空間即可。
   - **若使用 GitHub Pages**：上傳至 repo，並在 Settings > Pages 開啟發布即可。
   - **若使用自己的主機 (Apache/Nginx)**：直接放入 `/var/www/html` 或是您設定的 web root 目錄下即可。
2. 取得您新主機的前端網址（例如 `https://您的新網址.com`）。

### 3. 後台系統設定更新 (重要！)
1. 打開您剛剛部署好的新網址。
2. 點擊左下角進入 **「後台管理」**（預設密碼為 `admin123`）。
3. 點擊 **「系統設定」** 頁籤。
4. 將 **「系統對外網址 (Base URL)」** 欄位修改為您的「新主機網址」。
5. 點擊「儲存設定」。
   > [!IMPORTANT]
   > 必須完成此步驟，之後列印出來的標籤貼紙上面的 **QR Code** 才會指向您新的網址！

---

## ✅ 轉移完成驗證

完成上述步驟後，請進行以下測試以確保轉移成功：
1. **測量登記測試**：實際走一次測量流程，上傳一張測試照片，並確認是否能成功送出。
2. **資料庫確認**：開啟新的 Google 試算表，檢查 `測量紀錄` 工作表是否成功寫入剛剛的測試資料。
3. **雲端硬碟確認**：開啟您新建的相片資料夾，檢查照片是否成功上傳。
4. **後台登入測試**：確認後台可正常登入並查看到剛剛測試的紀錄。
